1 Interactuar con el sistema de archivos

1.2 Caminos de las carpetas

Los caminos o direcciones de las carpetas estn codificados como cadenas de carcter en R, siguen las normas de direcciones de los sistemas UNIX:

  • las carpetas hijas de la carpeta actual estn separadas por /
  • el signo . designa la carpeta actual
  • el signo .. designa la carpeta superior

En los proyectos, les aconsejo siempre utilizar caminos relativos, utilizando los signos . y .. para escribir las direcciones de los archivos. De est manera pueden mover la carpeta, y los codigos no deberan dejar de funcionar, si la relacin entre las carpetas de codigos y datos quedan iguales en la nueva direccin de carpeta. En Rstudio, Session>Set working directory>To source file location les permite aplicar un setwd a la carpeta donde est el script de R.

2 file encoding

El encoding es una definicin del set de caracteres utilizado en los archivos. Para evitar problemas de compatibilidad, les aconsejo utilizar siempre UTF-8, una norma unificada de encoding disponible en la mayora de los sistemas.

En R eso se puede hacer con:

Sys.setlocale(locale = 'es_CL.UTF-8')

Este comando se puede escribir en el archivo .Rprofile para que sea ledo antes de empezar cada sesin de R (ver https://support.posit.co/hc/en-us/articles/360047157094-Managing-R-with-Rprofile-Renviron-Rprofile-site-Renviron-site-rsession-conf-and-repos-conf).

En Rstudio, vale la pena mirar como poner UTF8 como el encoding por defecto (https://community.rstudio.com/t/how-can-i-change-the-default-encoding-when-sourcing/14350)

3 Archivos de texto

3.1 Leer archivos de texto

La mayora de los archivos que se utilizan son simplemente archivos de texto. Aunque tengan extensiones de archivos diferentes, los archivos txt, csv, o incluid los scripts de R, SQL o python son todos simplemente archivos de texto, lo que les permite estar ledo de manera nativa en todas las plataformas.

3.1.1 Funcin readlines

La funcin readLines es una de las funciones ms bsica para leer archivos de texto (existen unos mtodos para abrir archivos como conexiones de manera todava ms bsica, pero se necesitan raramente, los que quieren pueden mirar las funciones stdin, open, file etc). Lo que hace la funcin readLines es simplemente poner cada linea de un archivo de texto en un elemento de un vector de modo carcter.

readLines("../Datos/bog_chinga.csv")
[1] "\"UniMuestreo\",\"Especie\",\"abundancia\"" 
[2] "\"Bogot\",\"Canis lupus\",3000"            
[3] "\"Bogot\",\"Felis catus\",1000"            
[4] "\"Chingaza\",\"Tremarctos ornatus\",50"     
[5] "\"Chingaza\",\"Canis lupus\",100"           
[6] "\"Chingaza\",\"Odocoileus virginianus\",200"

4.2 Escritura

Para escribir un archivo xlsx, lo ms practico es utilizar la funcin createWorkbook

mi_xlsx<-createWorkbook()
addWorksheet(mi_xlsx,"programa")
addWorksheet(mi_xlsx,"estudiantes")
addWorksheet(mi_xlsx,"num_estudiantes")

Despus, podemos escribir los datos en cada worksheet

writeData(mi_xlsx,"programa",programa,rowNames = F)
writeData(mi_xlsx,"estudiantes",estudiantes,rowNames = F)
num_estu<-length(estudiantes)
writeData(mi_xlsx,"num_estudiantes",num_estu,rowNames = F)

Finalmente, podemos exportar el archivo excel con:

saveWorkbook(mi_xlsx,file="../Resultados/mi_xlsx", overwrite = T)

6 Archivos de datos para R

Exiten formatos de archivos de datos especificos para R: los archivos rds y RData.

Las diferencias entre esos archivos son:

  • rds se utiliza para guardar un solo objeto
  • RData se puede utilizar para guardar una lista de objetos
  • rds no contiene el nombre del archivo, se necesita afectar un nombre al importarlo
  • RData contiene los nombres de los objetos, as que los nombres no se pueden cambiar al importarlos

6.1 RData

Los archivos RData se guardan as:

save(list=c("programa","estudiantes"),file="../Resultados/cursoBiodata.RData")

y se importan as:

(load(file="../Resultados/cursoBiodata.RData"))
[1] "programa"    "estudiantes"

Nota: poner una linea de codigo entre parentesis permite imprimir el resultado en la consola, en el caso de load, es muy util porque contiene la lista de las variables importadas.

6.2 RDS

Los archivos RDS se guardan as:

saveRDS(object=num_estu,file="../Resultados/nbStud.rds")

y se importan as:

nbStud<-readRDS("../Resultados/nbStud.rds")
nbStud==num_estu
[1] TRUE
